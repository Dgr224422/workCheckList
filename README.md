# workCheckList

MVP-каркас Telegram-бота на **aiogram 3** для кинотеатра с раздельными БД по функциональным модулям.

## Что уже заложено

- Разделение данных по независимым БД:
  - `certificates.db` — сертификаты.
  - `popcorn.db` — ежедневный расход попкорна.
- Безопасный доступ к SQLite через параметризованные запросы (`?`), что снижает риск SQL-инъекций.
- Базовый стартовый интерфейс бота с меню разделов.
- Сервис декодирования QR-кода сертификата из изображения.
- Сервис расчёта расхода попкорна по заданным правилам:
  - 1.5 л — 40 вёдер в рукаве.
  - 3.0 л — 40 вёдер.
  - 6.0 л — 25 вёдер.
- Централизованное логирование в `logs/bot.log`.

### Дальше добавить (рекомендуемый roadmap)

1. **Сертификаты**
   - FSM-сценарий: ввод кода вручную / отправка фото с QR.
   - Неполный поиск по коду с выдачей кнопок-совпадений.
   - Статусы сертификата (активен/погашен/просрочен).

2. **Попкорн-отчёт**
   - FSM-диалог по 3 объёмам подряд (1.5, 3.0, 6.0).
   - Обязательное прикрепление фото отчёта.
   - Сверка с кассой и сообщение о расхождении.
   - Ротация данных старше 45 дней (удаление по дням).

3. **Рейтинг классов**
   - Отдельная БД `classes.db`.
   - Хранение телефона, школы, района, рейтинга, мест/рядов, фото нарушений.

4. **Чек-листы и напоминания**
   - Отдельная БД `checklists.db`.
   - Роли: system admin -> admin -> worker.
   - Шаблоны действий с отметкой "✅" и фото к шагам.
   - Планировщик напоминаний по дате и дню недели.

5. **График администратором на месяц**
   - Отдельная БД `schedule.db`.
   - Таблица смен + экспорт отчётов.

## Быстрый старт

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
# заполните BOT_TOKEN и SYSTEM_ADMIN_ID
python -m app.main
```

## Принципы безопасности

- Параметризованные SQL-запросы.
- Логирование действий пользователей и админов.
- Разделение прав по Telegram ID.
- Валидация форматов (телефон, даты, числовые поля).
- Именование файлов артефактов по шаблону: `YYYYMMDD_HHMMSS_<entity_code>.<ext>`.
